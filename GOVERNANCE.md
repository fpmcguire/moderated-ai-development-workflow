# Governance

## Overview

Moderated AI Development is an open-source methodology framework currently maintained by its author. This document describes how decisions are made and how the project is maintained.

## Maintainers

The project is currently maintained by its founder. The maintainer list may grow as the community grows. Maintainers are responsible for:

- Reviewing and merging pull requests
- Triaging issues
- Publishing releases and updating the CHANGELOG
- Enforcing the Code of Conduct

## Decision Making

### Minor Changes

Minor changes (typo fixes, clarifications, small template improvements) may be merged by any maintainer after a brief review.

### Significant Changes

Significant changes (new roles, revised ceremonies, updated quality gates, new prompt templates) require:

1. An open issue or discussion describing the proposed change
2. A pull request referencing the issue
3. At least one maintainer approval
4. A comment period of at least 5 business days for community feedback

### Breaking Changes

Changes that alter the core Moderated AI Development methodology (e.g., renaming core concepts, restructuring the step lifecycle) require:

1. A detailed proposal issue with rationale
2. An extended comment period (minimum 14 days)
3. Consensus among all active maintainers
4. A CHANGELOG entry and version tag
